### Step 4: AI Analysis & Qualification
Behind the scenes, NOVA:

```
1. Analyzes all responses against criteria:
   
   GOOD FIT (✅ Green light):
   └── Budget aligns with scope
   └── Project type matches TechNova capabilities
   └── Business model is viable
   └── User seems serious (legitimate use case)
   
   MEDIUM FIT (🟡 Caution):
   └── Budget is tight but doable
   └── May need to negotiate scope
   └── Could be good upsell later
   
   POOR FIT (❌ Red light):
   └── Budget unrealistic for scope
   └── Out of TechNova capability (e.g., hardware)
   └── Seems like "just curious" tire-kicker
   └── Use case is questionable

2. Scores lead qualification (0-100%):
   Factors:
   ├── Budget alignment (25%)
   ├── Project clarity (25%)
   ├── Timeline realism (20%)
   ├── Business viability (20%)
   └── Intent to proceed (10%)

3. Determines recommended product tier:
   ├── START tier: MVP, landing page, quick solutions
   ├── GROWTH tier: Full website, automation, scaling
   └── SCALE tier: Custom, complex, enterprise-level
```

### 3.3 Claude API Integration

NOVA uses Claude for intelligent analysis:

```
Prompt 1: INITIAL ANALYSIS
Input: User responses to initial questions
Task: 
  - Extract key information
  - Identify project category
  - Assess budget alignment
  - Flag any red flags

Example:
User responses:
- Name: Carlos
- Project: Artisan crafts e-commerce
- Products: 50 currently
- Customers: <100
- Budget: $4,000

Claude analyzes and outputs JSON:
{
  "category": "ecommerce",
  "budget": 4000,
  "scope_complexity": "medium",
  "project_clarity": "high",
  "red_flags": [],
  "confidence": 0.85,
  "recommended_tier": "growth",
  "estimated_cost": 6000,
  "budget_gap": -2000,
  "notes": "Budget is tight, may need to negotiate scope"
}

Prompt 2: FOLLOW-UP GENERATION
Input: Accumulated responses + analysis
Task:
  - Generate contextual follow-up questions
  - Ask what's unclear
  - Build on previous answers

Example:
Claude generates: "I noticed you have 50 products now but plan to add 200.
Let's talk about that:
- What's your timeline for adding those 200?
- Do you need bulk import tools?
- Should we build for 1,000+ products from day 1?"

Prompt 3: PROPOSAL GENERATION
Input: Complete questionnaire responses + analysis
Task:
  - Create custom project plan
  - Calculate timeline
  - Generate scope/deliverables
  - Build budget breakdown

Output: JSON proposal that renders on frontend

Prompt 4: QUALIFICATION DECISION
Input: All data + conversation history
Task:
  - Decide: Auto-checkout, manual review, or reject
  - Generate routing message
  - Flag for follow-up if needed

Output:
{
  "qualification_score": 78,
  "routing": "auto_checkout",
  "message": "Great news! You qualify for our standard process...",
  "follow_up_needed": false
}
```

## 📊 QUALIFICATION SCORING FORMULA

```
Score = (Budget × 0.25) + (Clarity × 0.25) + (Timeline × 0.20) 
        + (Viability × 0.20) + (Intent × 0.10)

Each factor scored 0-100:

Budget (0-100):
  Unrealistic for scope = 10
  Tight but doable = 50
  Perfect alignment = 80
  Above budget = 100

Clarity (0-100):
  Vague/confused = 10
  Some clarity = 50
  Clear but complex = 75
  Crystal clear = 100

Timeline (0-100):
  Unrealistic = 10
  Tight = 50
  Reasonable = 75
  Flexible = 100

Viability (0-100):
  Questionable business model = 10
  Unproven market = 50
  Established market = 75
  Clear market fit = 100

Intent (0-100):
  Just browsing = 10
  "Maybe someday" = 30
  "Considering" = 60
  "Ready to go" = 100

Result:
80-100 = AUTO CHECKOUT (✅ Go)
60-79 = MANUAL REVIEW (🟡 Vic decides)
<60 = REJECT (❌ Not a fit)
```
